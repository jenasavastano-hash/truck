# Деплой готов — что дальше

Сайт на VPS. Ниже — типовые шаги после первого выката.

---

## 1. HTTPS (сертификат)

Чтобы открывался **https://ваш-домен.example**:

1. Подключись по SSH к серверу.
2. Выполни (подставь свой домен):
   ```bash
   certbot --nginx -d ваш-домен.example -d www.ваш-домен.example
   ```
3. Введи email, согласись с правилами. Certbot сам настроит nginx и выдаст сертификат.
4. Сертификат будет продлеваться автоматически (таймер certbot).

---

## 2. DNS

Если домен ещё не открывается:

- В панели управления доменом в **A-записях** для корня и `www` укажи **IP вашего VPS**.
- Удали лишние A-записи с другими IP.
- Подожди 5–30 минут.

---

## 3. Первый вход

- Открой сайт (http или https).
- Если есть экран логина — нужен логин/пароль админа. Обычно они создаются через бэкенд (seed или первый пользователь). Проверь в коде/доках проекта, как создаётся первый админ.
- Для локальной разработки часто `admin` / пароль из .env или seed.

---

## 4. Обновления сайта

- **Полный деплой:** запусти **deploy.bat** → выбери **1** (Перезалить всё) → введи пароль.
- **Только фронт:** пункт **2**.
- **Только перезапуск бэка:** пункт **3**.
- **Сменился IP сервера:** пункт **4**, введи новый IP.

---

## 5. Если что-то сломалось

- Открой **deploy/CHECKLIST-СЕРВЕР.md** и пройди по шагам (бэкенд, nginx, права, фаервол).
- Логи бэкенда на сервере: `pm2 logs freight-epl-api` (или имя процесса из `pm2 list`).
- Логи nginx: `tail -50 /var/log/nginx/error.log`

---

## 6. Воркеры (медик, диспетчер, механик, QR) на ПК при API на сервере

Воркеры на твоём ПК **будут нормально работать**, если они ходят на API по адресу сервера, а не на localhost.

1. В папке **signer-client** открой или создай файл **.env**.
2. Задай:
   - **API_URL** — адрес сайта без слэша в конце, например: `https://ваш-домен.example` (или `http://IP_сервера` без HTTPS).
   - **SIGNER_API_KEY** — тот же ключ, что и в **backend/.env** на сервере (переменная `SIGNER_API_KEY` или `CLINIC_API_KEY`). Без него воркеры получат 401/503.

После этого при запуске воркеров они будут опрашивать `https://ваш-домен.example/api/clinic/...` и т.д. Логины Такском (TAKSKOM_DISPATCHER_*, TAKSKOM_MEDIC_*, TAKSKOM_MECHANIC_*) и профили браузеров остаются на ПК — так и задумано.

## Кратко

| Задача              | Действие |
|---------------------|----------|
| Включить HTTPS      | На сервере: `certbot --nginx -d ваш-домен -d www.ваш-домен` |
| Домен не открывается| Проверить A-записи DNS → IP VPS |
| Выкатить обновления | **deploy.bat** → 1 или 2 |
| Сменить IP сервера  | **deploy.bat** → 4 |
| Воркеры на ПК → API на сервере | В **signer-client/.env**: `API_URL=https://ваш-домен`, `SIGNER_API_KEY=<как на сервере>` |
