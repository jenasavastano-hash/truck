# Work Rules

## Общие принципы

- Любое изменение должно быть привязано к пункту [[04 - Roadmap]] и при необходимости отражено в [[08 - Tasks & Progress]] (чеклист или лог).
- Минимизируем “магические” поведения: важные решения документируем.
- Секреты не коммитим и не хардкодим (JWT, токены интеграций).

## Стиль кода

- Frontend: избегаем “god components”, выносим логику в `api/`, `hooks/`, `components/`.
- Backend: роуты тонкие, бизнес-логика в `services/`, утилиты в `utils/`.

## Обязательные артефакты на задачу

- Короткое описание изменения (что/почему)
- Изменения API (если есть) — в `05 - API Contracts.md`
- Мини-план тестирования (ручной чеклист)

## Правила “чистки”

- `node_modules/` и `dist/` не храним в проекте.
- Удаляем исходники только когда:
  - нет импортов/использований
  - нет роутов/ссылок
  - нет динамических `require()`/ленивых импортов
