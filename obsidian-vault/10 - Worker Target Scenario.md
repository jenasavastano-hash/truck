# Целевой сценарий оператора (worker) — черновик v1

Связано с [[09 - Worker Flow & Freight Domain]] и эпиком **WRK** в [[08 - Tasks & Progress]].

## Роль

**Оператор (worker)** — создаёт и ведёт ЭПЛ от имени парка/диспетчерской, видит статусы интеграций (Такском / QR / PDF), реагирует на ошибки и повторные попытки.

## Счастливый путь

1. Вход в кабинет оператора (отдельная роль или выделенный режим внутри менеджера — **решение зафиксировать**).
2. Список ЭПЛ: фильтры по парку, водителю, ТС, статусу, дате.
3. Создание черновика ЭПЛ → заполнение полей под **грузовую** перевозку (ТС, водитель, маршрут/рейс по правилам интеграции).
4. Отправка в обработку → ожидание статусов от внешней системы.
5. Успех: PDF/QR доступны водителю / в карточке ЭПЛ.
6. Закрытие смены/ЭПЛ по правилам парка.

## Отклонения

- Ошибка интеграции: явный статус + текст ошибки + действие **повторить** / **исправить данные**.
- Таймаут: не «висеть» без статуса — индикатор и ссылка на поддержку/лог (для админа).

## Не цель оператора

- Пассажирский заказ «как в такси» — вне домена; UI не ведёт по этой метафоре.
