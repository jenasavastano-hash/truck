# Воркеры и домен: пассажир → груз, рейс

## Тезис

1. **Переоформление работы воркеров (операторов)**  
   Сценарии, права и экраны должны соответствовать **созданию и ведению ЭПЛ в грузовом контексте**, а не «заказу поездки» или пассажирской логике. Имена полей, статусы, очереди ошибок — говорят на языке **рейса / ТС / путевого**, без такси-наследия там, где оно мешает.

2. **Выбор перевозки: с пассажирской на грузовую**  
   В коде и копирайте явно зафиксировать: продукт **не** про пассажиров; **грузовой рейс** — базовая метафора (маршрут, ТС, водитель, смена/ЭПЛ). Всё, что тянет к «пассажиру/поездке как в такси», выносится в отдельный список на удаление или замену.

## Что считать «готово по смыслу»

- Воркер видит очередь ЭПЛ и действия в терминах **ЭПЛ / статус / ТС / водитель**, без противоречий с бэкендом.
- Нет обязательных полей и подсказок только под пассажирский тариф (если не нужны по закону — убрать из UI).
- Документация оператора (внешние API) согласована с [[05 - API Contracts]].
