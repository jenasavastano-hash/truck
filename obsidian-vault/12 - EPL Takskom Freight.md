# Путевой лист: груз vs такси (Такском + воркер + PDF)

Скрины из ЛК Такском задают **целевой грузовой сценарий** (не легковое такси).

## Выбор при создании заявки (водитель)

В модалке «Новый путевой лист» (`DriverPortal`) водитель выбирает **вид коммерческой перевозки**; значение пишется в `epl.commercialShippingType` и уходит в очередь клиники/воркеру (`commercialShippingTaxcomLabel` в `GET /api/clinic/pending-creation`).

Коды и подстроки для ЛК — `backend/utils/commercialShippingTypes.js` (по умолчанию для грузового продукта — **ПГ**).

## API (`backend/takskom-api.js`, `createSimpleEPL`)

| Поле API | Было (такси) | Сейчас по умолчанию (груз) | Переопределение |
|----------|----------------|---------------------------|-----------------|
| `commercialShippingType` (при `shippingType=КП`) | `ЛТ` | `ПГ` (перевозка грузов по договору — уточните по договору с Такском) | `TAKSKOM_COMMERCIAL_SHIPPING_TYPE` в `backend/.env` или `parkInfo.commercialShippingType` |

Остальное (`tripFeature`, `shippingType`, `areaShippingType`, …) по-прежнему из данных парка / дефолтов в коде — при необходимости расширим полями парка в БД.

## Воркер Playwright (`signer-client/taxcom-create.js`)

После признака «один день» и медосмотра вызывается `applyFreightWaybillTaxcomFields`:

- **Вид коммерческой перевозки** — option, в тексте есть подстрока из `TAXCOM_COMMERCIAL_SHIPPING_LABEL` (по умолчанию `перевозка грузов`).
- **Признак начала рейса** — `TAXCOM_TRIP_START_LABEL` (по умолчанию `парковк` → «выезд с парковки…»).
- **Сведения о виде перевозки** — `TAXCOM_SHIPPING_KIND_LABEL` (`коммерческ` → коммерческие перевозки).
- **Сведения о виде сообщения** — `TAXCOM_MESSAGE_KIND_LABEL` (`городск` → городское).

Селекторы в коде — несколько запасных `name`/`id`; если у вас другая вёрстка ЛК, смотрите реальные `select` в DevTools и при необходимости добавьте селектор в код или уточните подстроку.

**Тип ТС** в поле `#vehicle_type`: из карточки авто (`cars.vehicleType`) или `TAXCOM_DEFAULT_VEHICLE_TYPE` / **`грузовой`**.

**Послесменный медосмотр**: по умолчанию **да** (`value` 1). Старый режим «нет»: `TAXCOM_POST_SHIFT_MEDICAL_REQUIRED=0`.

Переменные см. `signer-client/.env.example`.

## Быстрый PDF (`worker_ep_l.py` + `fast-epl-pdf.js`)

Строка «Тип: …» в PDF строится из **`commercialShippingType` заявки** и карточки ТС: для **ПГ** (перевозка грузов) в бланке всегда **«грузовой»**; для РП/ЗП/ТЛ — «легковой», для ОД — «автобус» (`buildVehicleLineForFastPdf` в `fast-epl-pdf.js`).

## Адреса погрузки / выгрузки (несколько точек)

На скринах — отдельные блоки адресов и кнопки «Добавить». **Автозаполнение в воркере пока не подключено** (нужны стабильные id полей в вашей версии ЛК и сценарий маршрута). Дальше: либо расширение Playwright-шагов, либо передача точек из заявки (новые поля в API заявки).

## Проверка

1. В `signer-client/.env` задать подстроки при расхождении с ЛК.
2. Создать тестовую заявку и пройти форму с `SCREENSHOTS=1` / пошаговой отладкой.
3. Убедиться, что в выгрузке PDF/ГИС вид **перевозка грузов**, а не легковое такси.
