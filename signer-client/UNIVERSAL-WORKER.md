# Универсальный воркер

Один процесс обрабатывает роли: **диспетчер** (Т1), **медик** (Т2), **механик** (Т3/Т4).  
Проверяет статусы титулов перед обработкой. QR не входит — при необходимости запускайте `qr-fetcher.js` отдельно.

## Как запустить

### Из панели (рекомендуется)
1. Остановите старые воркеры (закройте окно «Workers» или выберите в панели пункт 4 и не запускайте 5).
2. `node dev-control.js` → **4) Воркеры** → **6) Универсальный**.
3. Откроется окно с одним процессом. Логи пишутся в консоль и в `signer-client/logs/universal-worker-YYYY-MM-DD.log`.

### Из командной строки
```bash
cd signer-client
set USE_UNIVERSAL_WORKER=1
node run-workers-with-pid.js
```
Или только универсальный воркер (без PID-файла для панели):
```bash
cd signer-client
node universal-worker.js
```
Один тик (для проверки):
```bash
node universal-worker.js --once
```

### Через npm
```bash
cd signer-client
npm run universal
```

## Как проверить

1. **Запуск**: убедитесь, что в консоли есть строки вида `[dispatcher] Браузер открыт`, `[medic] Браузер открыт`, `[mechanic] Браузер открыт`.
2. **Опрос**: каждые N секунд (см. `POLL_SECONDS` в `.env`) выводится строка «Заявок: X (дисп.: …, медик: …, механик: …)».
3. **Обработка**: при появлении заявок в логах появятся сообщения `[Dispatcher] Заявка: …`, `[Medic] Заявка: …`, `[Mechanic] Заявка: …` и результат (ЭПЛ создан / Т2 подписан / Т3 и Т4 подписаны).
4. **Логи**: смотрите `signer-client/logs/universal-worker-<дата>.log`.
5. **Остановка**: в окне воркера нажмите **Ctrl+C**.

## Переменные .env

Те же, что для отдельных воркеров: `API_URL`, `SIGNER_API_KEY`, `TAKSKOM_DISPATCHER_PHONE`, `TAKSKOM_MEDIC_PHONE`, `TAKSKOM_MECHANIC_PHONE` и пароли, `POLL_SECONDS`. Опционально: `BROWSER_TIMEOUT_MS` (таймаут одной операции в браузере, по умолчанию 5 мин).
