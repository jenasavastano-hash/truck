# Настройка КриптоПро для автоподписи ЭПЛ

При нажатии «Подписать» в ЛК Такском должен открываться диалог КриптоПро (выбор сертификата и подпись). Если диалог не появляется — причина почти всегда в том, что **скрипт запускает браузер без расширения**.

## В браузере нет КриптоПро — что сделать

**Яндекс.Браузер:** в `.env` поставь `USE_YANDEX_BROWSER=1`. Путь к браузеру и профиль подставятся сами. Установи расширение КриптоПро вручную в Яндекс.Браузере, затем **полностью закрой браузер** (все окна + заверши процесс в диспетчере задач, если есть «Яндекс.Браузер» в фоне) — и только потом запускай `node sign.js`. Иначе ошибка «Target page, context or browser has been closed» или «профиль занят».

**ГОСТ из папки:** укажи в `CHROMIUM_GOST_PATH` путь к chrome.exe из папки, где при ручном запуске есть КриптоПро. Скрипт возьмёт профиль «User Data» рядом с exe. Пример: `CHROMIUM_GOST_PATH=D:\Chromium-GOST\chrome.exe`.

Скрипт использует браузер и его профиль — расширение КриптоПро должно быть установлено в **этом** браузере.

1. **Закрой все окна Chromium** (тот, что в `CHROMIUM_GOST_PATH`).
2. **Установи расширение КриптоПро ЭЦП для браузера:**
   - Скачай установщик плагина с сайта КриптоПро: [cryptopro.ru — КриптоПро ЭЦП Browser plug-in](https://www.cryptopro.ru/products/cades/plugin/get_2_0) или [gost.cryptopro.ru](https://gost.cryptopro.ru/).
   - Запусти установщик и выбери установку для **Chrome/Chromium** (если спрашивает).
   - Либо: запусти **этот же Chromium** вручную, открой `chrome://extensions/`, включи «Режим разработчика», «Загрузить распакованное расширение» — укажи папку, куда установщик положил расширение (или распакуй из дистрибутива КриптоПро).
3. **Закрой Chromium полностью** (все окна).
4. Запусти снова `node sign.js` — при нажатии «Подписать» в Такскоме должен появляться КриптоПро.

**Важно:** на ПК должен быть установлен **КриптоПро CSP** ([gost.cryptopro.ru](https://gost.cryptopro.ru/)); без него подпись не работает. Расширение из Chrome Web Store могли удалить — ставь с сайта КриптоПро. Если используешь **Chromium-GOST** из сборки под ГОСТ — в нём расширение может быть уже включено.

**Автоподпись без диалога (рекомендуется):** добавь Такском в доверенные сайты КриптоПро — тогда диалог «Подтверждение доступа» не будет появляться и подпись пойдёт автоматически. **Пуск → КРИПТО-ПРО → Настройки ЭЦП Browser plug-in** → открой список доверенных сайтов → добавь **`https://epl.taxcom.ru`** (или `https://epl.taxcom.ru/`). После этого скрипт сможет подписывать титулы без нажатия «Да» вручную и без автоклика по окну.

**Диалог «Подтверждение доступа» (нативное окно):** если сайт не в доверенных, КриптоПро показывает «Разрешить эту операцию?» (Да/Нет). Скрипт может сам нажимать «Да»: в фоне запускается PowerShell (UI Automation). Отключить автоклик: в `.env` задай `CRYPTOPRO_AUTO_CLICK_ALLOW=0`.

## В чём проблема

Playwright по умолчанию запускает **чистый** браузер без расширений и без твоего профиля Chrome. Расширение **КриптоПро ЭЦП** (для браузера) в таком окне не загружается, поэтому при нажатии «Подписать» КриптоПро не вызывается.

## Что сделать: профиль Chrome с расширением

Нужно запускать **Chrome с отдельным профилем**, в котором один раз установлено расширение КриптоПро.

### 1. Установить КриптоПро на ПК

- Установи **КриптоПро CSP** (лицензия/триал): [gost.cryptopro.ru](https://gost.cryptopro.ru/) или [cryptopro.ru](https://cryptopro.ru).
- Установи **расширение КриптоПро ЭЦП для браузера** (для Chrome) с сайта КриптоПро или из Chrome Web Store (КриптоПро ЭЦП Browser plug-in).

### 2. Создать профиль для автоматизации

1. Создай пустую папку для профиля, например:
   - `C:\chrome-epl-profile`
   - Или `C:\Users\Данил\AppData\Local\chrome-epl-signer`

2. **Один раз** запусти Chrome с этой папкой как профилем:
   - Закрой все окна Chrome.
   - Запусти Chrome с указанием профиля (в командной строке):
   ```bat
   "C:\Program Files\Google\Chrome\Application\chrome.exe" --user-data-dir=C:\chrome-epl-profile
   ```
   Если используешь Chromium-Gost (путь из `.env`):
   ```bat
   "C:\Users\Данил\AppData\Local\Chromium\Application\chrome.exe" --user-data-dir=C:\chrome-epl-profile
   ```

3. В открывшемся окне:
   - Зайди в [Chrome Web Store — КриптоПро ЭЦП](https://chrome.google.com/webstore) и установи расширение КриптоПро ЭЦП для браузера (или с сайта КриптоПро).
   - При необходимости войди в ЛК Такском и проверь, что в настройках профиля/подписания выбран способ **«КриптоПро»**, а не только «мобильное приложение».
   - Закрой Chrome.

### 3. Указать профиль в signer-client

В папке `signer-client` в файле **`.env`** добавь (или измени):

```env
TAXCOM_USER_DATA_DIR=C:\chrome-epl-profile
```

Путь — тот же, что в `--user-data-dir` (полный путь к папке без слэша в конце).

После этого при запуске `node sign.js` скрипт будет открывать Chrome **с этим профилем** — расширение КриптоПро загрузится, и при нажатии «Подписать» должен появляться диалог КриптоПро.

### 4. Дополнительно

- **TAXCOM_HEADLESS=0** — браузер видим (для отладки). При заданном `TAXCOM_USER_DATA_DIR` браузер и так запускается в режиме «не headless».
- **CHROMIUM_GOST_PATH** — путь к Chrome/Chromium-Gost; тот же exe будет использоваться при запуске с профилем.
- Если диалог КриптоПро всё равно не появляется: проверь в обычном Chrome (с тем же профилем), что при нажатии «Подписать» на сайте Такском открывается КриптоПро. Если там не открывается — настраивать нужно браузер/расширение/КриптоПро CSP, а не скрипт.

## Кратко

| Проблема | Решение |
|----------|--------|
| КриптоПро не вызывается при «Подписать» | Запускать Chrome с профилем, в котором установлено расширение КриптоПро. Задать в `.env`: `TAXCOM_USER_DATA_DIR=C:\путь\к\папке\профиля`. |
| Где взять расширение | КриптоПро ЭЦП Browser plug-in (Chrome) с сайта КриптоПро или Chrome Web Store. |
| В профиле только расширение | Достаточно: создать папку → один раз запустить Chrome с `--user-data-dir=эта папка` → установить расширение → закрыть Chrome → указать путь в `TAXCOM_USER_DATA_DIR`. |
