# Установка сертификатов с закрытым ключом для подписания ЭПЛ

## Проблема

У тебя установлены `.cer` файлы, но они **не содержат закрытый ключ**. Без закрытого ключа подписание невозможно.

**Статус в диагностике:** "Нет привязки к закрытому ключу" ❌

## Решение

### Вариант 1: Установка через .pfx файлы (рекомендуется)

Если у тебя есть `.pfx` файлы (с закрытым ключом):

1. **Открой КриптоПро CSP:**
   - Панель управления → КриптоПро CSP → Сертификаты
   - Нажми "Установить сертификат"

2. **Выбери .pfx файл** и введи пароль (если требуется)

3. **Выбери хранилище:** "Личное" (My)

4. **Повтори для всех сертификатов:**
   - Амиргамзаев.pfx (диспетчер)
   - Поливода.pfx (медик)
   - Иванов.pfx (механик)
   - Таов Инал Нарзанович.pfx (если нужен)

### Вариант 2: Установка через командную строку

Если у тебя есть `.pfx` файлы, можно установить через командную строку:

```cmd
# Установка .pfx файла в хранилище "Личное"
certutil -importPFX -f -user My "путь\к\файлу.pfx"
```

**Пример:**
```cmd
certutil -importPFX -f -user My "C:\Users\Данил\Desktop\Сайт Такси1\Амиргамзаев.pfx"
```

### Вариант 3: Получение .pfx из .cer (если есть доступ к закрытому ключу)

Если у тебя есть доступ к закрытому ключу, но он не привязан к `.cer`:

1. **Открой КриптоПро CSP:**
   - Панель управления → КриптоПро CSP → Сертификаты

2. **Найди сертификат** в списке (который нужно экспортировать с ключом)

3. **Экспортируй сертификат с закрытым ключом:**
   - Правой кнопкой → "Экспортировать"
   - Выбери "Да, экспортировать закрытый ключ"
   - Сохрани как `.pfx` файл

4. **Установи .pfx файл** (см. Вариант 1 или 2)

### Вариант 4: Получение новых сертификатов

Если у тебя нет доступа к закрытым ключам:

1. **Получи новые сертификаты** с закрытым ключом:
   - Через УЦ Такском: https://aist.taxcom.ru/
   - Или через другой удостоверяющий центр

2. **При получении сертификата:**
   - Сохрани его как `.pfx` файл (с закрытым ключом)
   - Запомни пароль от `.pfx` файла

3. **Установи .pfx файлы** (см. Вариант 1 или 2)

## Проверка установки

После установки `.pfx` файлов:

1. **Открой страницу диагностики:**
   - https://cryptopro.ru/sites/default/files/products/cades/demopage/cades_bes_sample.html

2. **Выбери сертификат** в списке

3. **Проверь статус:**
   - ✅ Должно быть: "Статус: Есть привязка к закрытому ключу"
   - ❌ Не должно быть: "Статус: Нет привязки к закрытому ключу"

4. **Попробуй подписать:**
   - Введи текст "Hello World"
   - Нажми "Подписать"
   - Должно появиться окно ввода пароля (если ключ защищён паролем)
   - После подписания должна появиться подпись в формате base64

## Скрипт для установки .pfx файлов

Создай файл `install-pfx-certificates.js` в папке `signer-client`:

```javascript
const { execSync } = require('child_process');
const fs = require('fs');
const path = require('path');

const projectRoot = path.join(__dirname, '..');
const certDir = __dirname;

console.log('[Сертификаты] Ищу .pfx файлы...');

// Ищем .pfx файлы
const pfxFiles = [];
const searchPaths = [projectRoot, certDir, path.join(projectRoot, 'certificates')];

for (const searchPath of searchPaths) {
  if (fs.existsSync(searchPath)) {
    const files = fs.readdirSync(searchPath).filter(f => f.toLowerCase().endsWith('.pfx'));
    for (const file of files) {
      const fullPath = path.join(searchPath, file);
      if (!pfxFiles.find(c => c.path === fullPath)) {
        pfxFiles.push({ name: file, path: fullPath });
      }
    }
  }
}

if (pfxFiles.length === 0) {
  console.error('[Сертификаты] ❌ Не найдено ни одного .pfx файла!');
  console.error('[Сертификаты] Положи .pfx файлы в корень проекта или в папку signer-client/');
  process.exit(1);
}

console.log(`[Сертификаты] Найдено .pfx файлов: ${pfxFiles.length}`);
pfxFiles.forEach(c => console.log(`  - ${c.name}`));

// Функция установки .pfx файла
function installPFX(certPath, certName) {
  try {
    console.log(`[Сертификаты] Устанавливаю ${certName}...`);
    console.log(`[Сертификаты] ⚠ ВНИМАНИЕ: Если потребуется пароль, введи его вручную!`);
    
    // certutil -importPFX -f -user My "путь_к_файлу.pfx"
    const command = `certutil -importPFX -f -user My "${certPath}"`;
    const output = execSync(command, { encoding: 'utf8', stdio: 'pipe' });
    
    console.log(`[Сертификаты] ✓ ${certName} установлен успешно`);
    return true;
  } catch (error) {
    console.error(`[Сертификаты] ❌ Ошибка при установке ${certName}:`, error.message);
    if (error.stdout) console.log('[Сертификаты] Вывод:', error.stdout);
    if (error.stderr) console.error('[Сертификаты] Ошибка:', error.stderr);
    return false;
  }
}

// Устанавливаем все найденные .pfx файлы
let successCount = 0;
for (const cert of pfxFiles) {
  if (installPFX(cert.path, cert.name)) {
    successCount++;
  }
}

console.log(`\n[Сертификаты] Установлено: ${successCount} из ${pfxFiles.length}`);

if (successCount === pfxFiles.length) {
  console.log('[Сертификаты] ✅ Все сертификаты установлены успешно!');
  console.log('[Сертификаты] Теперь можно использовать их для подписания ЭПЛ.');
} else {
  console.warn('[Сертификаты] ⚠ Некоторые сертификаты не удалось установить.');
  console.warn('[Сертификаты] Проверь права доступа и наличие паролей.');
}
```

**Запуск:**
```cmd
cd signer-client
node install-pfx-certificates.js
```

## Важно

- ✅ **Расширение КриптоПро работает отлично** в Chromium-Gost
- ✅ **Сертификаты найдены** в хранилище Windows
- ❌ **НО:** нужны сертификаты **с закрытым ключом** (`.pfx` файлы)

Без закрытого ключа подписание **невозможно**!
