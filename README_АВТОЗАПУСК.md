# Автоматический запуск проекта

## Быстрый старт

### Вариант 1: Полная автоматизация (рекомендуется)

1. **Создай файл `.env` в корне проекта:**
   ```env
   NETLIFY_SITE_ID=твой-site-id
   NETLIFY_API_TOKEN=твой-api-token
   ```

2. **Как получить Site ID:**
   - Зайди в Netlify Dashboard
   - Site settings → General → Site details → Site ID
   - Или просто скопируй из URL: `app.netlify.com/sites/[SITE_ID]`

3. **Как получить API Token:**
   - User settings (твой профиль справа вверху)
   - Applications → New access token
   - Создай токен с правами на изменение сайта
   - Скопируй токен (показывается только один раз!)

4. **Запусти:**
   ```
   АВТОМАТИЧЕСКИЙ_ЗАПУСК.cmd
   ```

   Скрипт автоматически:
   - ✅ Запустит бэкенд
   - ✅ Запустит Cloudflare Tunnel
   - ✅ Получит URL туннеля
   - ✅ Обновит переменную `VITE_API_URL` на Netlify
   - ✅ Запустит передеплой

### Вариант 2: Упрощённый (без Netlify API)

Просто запусти:
```
АВТОМАТИЧЕСКИЙ_ЗАПУСК_ПРОСТОЙ.cmd
```

Скрипт запустит всё, но нужно будет вручную:
- Скопировать URL из окна Cloudflare Tunnel
- Обновить переменную на Netlify

## Что делает скрипт

1. **Запуск Backend** - стартует на `http://localhost:5000`
2. **Запуск Cloudflare Tunnel** - создаёт публичный URL
3. **Получение URL** - парсит вывод Cloudflare и извлекает URL
4. **Обновление Netlify** - через API обновляет переменную окружения
5. **Передеплой** - запускает новую сборку на Netlify

## Устранение проблем

### Скрипт не находит URL туннеля
- Подожди немного (туннель создаётся до 30 секунд)
- Проверь окно Cloudflare Tunnel - там должен быть URL
- Скопируй URL вручную и обнови Netlify

### Ошибка обновления Netlify
- Проверь что `NETLIFY_SITE_ID` и `NETLIFY_API_TOKEN` правильные
- Убедись что токен имеет права на изменение сайта
- Обнови вручную через Dashboard

### Бэкенд не запускается
- Проверь что порт 5000 свободен
- Убедись что зависимости установлены: `cd backend && npm install`
- Проверь логи в окне Backend Server

## Структура файлов

- `АВТОМАТИЧЕСКИЙ_ЗАПУСК.cmd` - полная автоматизация с Netlify API
- `АВТОМАТИЧЕСКИЙ_ЗАПУСК_ПРОСТОЙ.cmd` - упрощённый вариант
- `.env` - переменные окружения (создай сам)
- `.env.example` - пример файла с переменными

## Безопасность

⚠️ **Важно:** Не коммить файл `.env` в git! Он содержит секретные токены.

Добавь в `.gitignore`:
```
.env
tunnel_output.log
```
